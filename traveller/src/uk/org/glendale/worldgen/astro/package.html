The uk.org.glendale.worldgen.astro package holds the entity definitions and logic for all astronomical objects within the universe. This includes sectors, star systems, stars and planets.

Each type of entity has its own package, which includes entities, factories, generators and API classes. The format of each package is similar for each, and code has been split across several classes to try and reduce the maintenance headache.

Entity

The entity class describes the meta data for the entity, and contains only a minimal amount of logic (if any). The idea is to keep these as simple as possible. They are named for the entity.

API

These are named as <entityt>API, and contain the logic for the entity during its lifetime, as well as declaring public methods for controlling the entity. Web Services are defined in here using javax.ws.* annotations.

Factory

These are named as <entity>Factory, and contain the logic for fetching existing instances of this entity.

Generator

These are named as <entity>Generator, and contain the logic for creating new instances of this entity. Creation logic is separated from fetching logic because for many of these entities, creation is very complex but happens rarely. By splitting it from the Factory class, we allow the Factory class (which is used frequently) to be simpler.
